Status note. This item is not a design exercise — the change is fully proposed (openspec/changes/improve-musicality/proposal.md, design.md, tasks.md, and two spec deltas under specs/show-orchestration/ and specs/show-refinement/). This document is a faithful digest of that design plus execution guidance: exact code seams, sequencing constraints, and per-phase acceptance criteria. Where this document and the OpenSpec change disagree, the change wins.

1. Why

The proposal's diagnosis (proposal.md "Why"): generated shows are technically correct but read as samey and scattered next to a hand-sequenced show, and the causes are structural, not tuning:

  • One recipe everywhere. generate.realize_section applies the same stack — bed + weave fabric + beat accents + sparkle contrast — to every section; only brightness and coverage scale with intensity. A quiet verse is "the chorus, but dimmer"; nothing is ever truly absent, so the peak has no contrast to land against. Coverage QA reinforces this by treating darkness as an objective error regardless of section energy.
  • Variety is keyed to position, not musical identity. weave.section_carrier(seed) rotates the carrier by section index, so chorus 1 and chorus 2 — which MusicBrief.repetition_map knows are the same music — get different carriers, and different peak composites via _PEAK_COMPOSITES[si % n]. The pipeline injects difference exactly where a sequencer would enforce sameness; repeated music never visually rhymes. Today repetition_map is used only to bump intensity (beats.effective_intensity).
  • Section boundaries are butt joints. The harmony analyst emits transition_cues_ms (music_brief.py:62) and the Director writes a SectionPlan.transition field (show_plan.py:40), but nothing downstream consumes either. Hand shows live on risers into choruses, a one-beat blackout before the drop, and sweep handoffs.
  • No show-level color story, no phrase-level dynamics. Palettes are per-section with no cross-section constraint; brightness value curves fire only on >15 s washes at intensity ≥ 0.7 (generate.py:172), so nothing swells or decays with the music's 2/4/8-bar phrasing.
  • The refine loop can't protect any of this because no metric measures it — what isn't measured gets optimized away.

2. Current state

2.1 Position-keyed variety

pipeline/weave.py:198–216 — the carrier rotation and its per-index keying:

CARRIER_ROTATION = ("SingleStrand", "Bars", "Garlands", "Wave")

def section_carrier(seed: int) -> str:
    """The rotated carrier effect for a section index (deterministic; cycles CARRIER_ROTATION)."""
    return CARRIER_ROTATION[seed % len(CARRIER_ROTATION)]

pipeline/generate.py:184–187 calls it with the section index: carrier = section_carrier(si), then diversify_carrier(weave_obj, carrier) swaps any "plain" LLM carrier (chase family + On, _ROTATABLE_CARRIERS at weave.py:201) to the rotated one. Distinctive LLM-chosen carriers (Spirals, Pinwheel…) are preserved — an invariant Phase 1 must keep.

pipeline/generate.py:55 and :196–199 — the peak composite is also index-keyed:

_PEAK_COMPOSITES = ("kaleidoscope", "swirl", "bloom", "ember")
...
if si in _peaks and HERO_GROUP in st.available_groups:
    cc = curated_composite(_PEAK_COMPOSITES[si % len(_PEAK_COMPOSITES)], [HERO_GROUP])

Per-effect palettes rotate by instruction index (beats.effect_palette(section_palette, effect_type, index), beats.py:74–82), and the expanded-palette order is therefore also position-derived.

2.2 Escalation is gain-only

pipeline/beats.py:127–138:

def escalation_level(section_index: int, repetition_map: dict | None) -> float:
    """0 for the first occurrence of a recurring section → 1 for the last; 0 if it doesn't recur."""
    ...

def effective_intensity(intensity, section_index, repetition_map) -> float:
    """Section intensity lifted by its escalation level so later recurrences build."""
    return min(1.0, base + ESCALATION_BOOST * escalation_level(...))

ESCALATION_BOOST = 0.25 (pipeline/tuning.py). Escalation today is brightness/speed/coverage-cap via intensity only — the chorus gets louder, not structurally bigger. coverage_cap(intensity, n_groups) (beats.py:141–144) and trim_coverage are the coverage lever the change will spend ordinally.

2.3 Coverage QA punishes deliberate darkness only above 0.5

qa/coverage.py:13–16: MIN_INTENSITY = 0.5, EXPECTATION = 0.6, FAIL_FRACTION = 0.5. Sections below 0.5 intensity are exempt; at/above it the expected lit fraction is a flat EXPECTATION × intensity against the show's own peak. There is no treatment concept, so a deliberately sparse mid-energy "feature" section (say intensity 0.55 with one hero prop lit) is an objective error today.

2.4 SectionPlan and the Director

show_plan.py:26–51 — SectionPlan has transition: str = "" (unconsumed) and phrasing (consumed by resolve_phrasing), but no treatment field. agents/director.py::render_input (lines 23–90) builds the Director prompt; it already carries per-section doctrine blocks (phrasing, scenes, feature-prop pop) — the treatment doctrine block lands in this string. director.section_redesigner() reuses the same prompt for mid-refine section redesigns, so treatment guidance automatically applies there too.

2.5 The two seams (why this change is tractable)

design.md "Context": since the consolidation, all per-section realization flows through generate.realize_section (generate.py:146–219) — shared by first-pass generation (generate_instructions), the refine loop (run.py::regenerate_section, line 124), and xlo regen — and all whole-list passes flow through generate.finalize_effects (generate.py:222–231). That gives exactly two seams: treatments/identity/dynamics inside realize_section; transitions/color-script as list-level passes before finalize_effects. Every needed signal is already computed and cached: repetition_map, transition_cues_ms, per-section intensity, energy_arc, beats/downbeats via section_rhythm, stems, key moments. No new audio analysis.

2.6 The advisory channel the new metrics ride

refine.Finding has objective: bool = True — "objective gates the loop; advisory only informs" (refine.py:17). qa/__init__.evaluate (lines 16–35) averages only sync/placement/rules(/coverage when a sampler exists) into objective_score; advisory findings ride findings into the Judge (judge_mod.render_input(report, …) at run.py:229) and into the revision log (run.py:201–209, _record maps f.metric through revision_log.source_of). qa/variety.py is the house style for advisory metrics: bounded findings, distinct metric names (its group-coverage vs objective coverage rename exists precisely because the revision log couldn't tell them apart — the new metrics must use distinguishable names from day one: musicality:rhyme, musicality:dynamic-range, musicality:focus).

3. Proposed design (digest of design.md)

Three phases; each lands independently and is valuable alone. Governing principles (design.md Goals): all new QA metrics are advisory (objective=False) so the refine loop's convergence properties are untouched; deterministic and code-owned wherever possible — the LLM chooses intent (treatment, palette direction), code realizes it. Non-goals: no analyzer schema changes, no emitter/layer-budget/occlusion changes, no lyric-word-level effects, no new Director round-trips for the color script.

3.1 Phase 1 — repetition identity (theme & development)

Decision 1 — identity keying: label hash, index fallback. New section_identity(si, repetition_map) -> str | None (in beats.py, beside escalation_level): the label whose indices contain si, counted only when that label recurs. section_carrier and the peak-composite pick key their rotation on hash(label) % len(rotation) when an identity exists, else on the index as today. The expanded-palette rotation order is keyed per label the same way (the per-effect index interplay inside a section stays as-is). One-off sections keep today's index-keyed variety. Zero schema change.

Decision 2 — escalation spends structurally. Beyond the existing effective_intensity lift: occurrence ordinal raises coverage_cap by +1 group per step, tightens the accent stride (_off_beat_stride) one step, and the final occurrence adds one extra layer (the sparkle/feature contrast layer when the layout has accent props, i.e. ACCENT_GROUPS ∩ available_groups). Bounded by the existing clamp_hard_caps (qa/rules.py:137) and clamp_layer_budget (effect_emitter.py:40) so it cannot blow the budget.

New advisory metric — repetition-rhyme (Decision 8, first metric): for each recurring label, Jaccard similarity of (target, effect_type) sets between occurrences, plus carrier equality; score = mean over labels. Emits at most one finding per label. A pre-change (index-rotated) show scores low; the point is that the Judge sees when a repeat doesn't rhyme, and the revision log can watch the metric across runs.

Spec anchor (specs/show-orchestration/): "Choruses share a carrier" — repetition-map siblings receive the same weave carrier, composite, and palette rotation; "The last chorus is the biggest" — the final occurrence lights at least as many groups, denser accents, +1 layer, within budgets.

Touched modules: pipeline/beats.py (new section_identity), pipeline/weave.py (section_carrier keying), pipeline/generate.py (thread the label; peak-composite + palette-order keying; structural escalation in realize_section), new qa/musicality.py + wiring in qa/__init__.py. Risk profile: visual change, no schema change — golden churn is expected and deliberate.

3.2 Phase 2 — section treatments (texture contrast)

Decision 3 — treatments are a realization contract, not a prompt suggestion. SectionPlan.treatment: Literal["full","feature","pulse","gesture","rest"] | "" = "" — defaulted so old cached plans still validate. The Director prompt gains a short doctrine block ("choose per section; withhold, don't just dim"). Deterministic fallback when empty (resolve_treatment(section, peaks)): intensity ≥ peak-band → full; ≥ 0.5 → pulse; ≥ 0.25 → feature if the layout has a focal prop else pulse; < 0.25 → gesture; explicit near-zero (< 0.1) → rest. realize_section includes layers per treatment:

treatmentbedweaveaccentscomposites/VUfeature
full✓✓✓✓✓
pulse✓—✓—✓
featuredim—sparse—✓ (hero)
gesture—one carrier recipe on ≤2 groups———
restdim, ≤2 groups————

Hard floor against a near-black show: at most 2 consecutive non-full/pulse sections get a bed injected (design.md Risks). Note the existing restraint machinery (section_is_rhythmic, rhythm_pool returning [] for quiet briefs) already withholds some layers; treatments make withholding an explicit, named contract rather than an emergent property of empty pulse_groups.

Decision 4 — coverage QA keys expectations to energy + treatment. qa/coverage.py scales its expected-lit threshold by section intensity (roughly the coverage_cap curve), and gesture/rest sections are exempt from the darkness error entirely (advisory note instead). This is the single QA change with objective-score impact, and it loosens only — existing passing shows keep passing. Spec anchor (specs/show-refinement/): "Deliberate darkness passes / a broken bright section still fails."

New advisory metrics — dynamic-range and focus-budget (Decision 8): dynamic-range = normalized spread between the min and max section of mean(brightness × lit-group-fraction), computed on instructions; low spread → "wall-to-wall brightness" finding. Focus-budget = per-section peak count of concurrent distinct moving-effect systems (qa/rules.MOTION_EFFECTS targets grouped by effect_type), flagged when it exceeds an energy-keyed budget (quiet: 1, mid: 2, peak: 3 + bed).

Touched modules: show_plan.py (schema field), agents/director.py (doctrine block), brief_editor.py (add treatment to the form's field ORDER list, brief_editor.py:76), pipeline/generate.py (treatment branching in realize_section), qa/coverage.py, qa/musicality.py. Risk profile: adds a schema field (defaulted, backward compatible with cached creative_brief JSON) + a behavioral QA loosening; the riskiest phase because it touches the coverage gate and the same realize_section seams the refine loop drives — hence the I3 sequencing constraint below.

3.3 Phase 3 — connective tissue (transitions, color script, phrase dynamics)

Decision 5 — transitions are an instruction-list pass, tagged to the outgoing section. New pipeline/transitions.py::place_transitions(st, instrs) -> list[EffectInstruction], run in generate_instructions and after refine-loop/xlo regen splices, before finalize_effects. Detection is pure signal math from energy arc + downbeats + transition_cues_ms:

  • boundary energy delta ≥ riser threshold → a 2-bar brightness-ramp chase on a broad group ending at the boundary;
  • delta ≥ drop threshold with a downbeat landing → gate (trim) all instructions in the final beat before the boundary — the blackout-before-the-hit;
  • otherwise lateral → optional sweep handoff.

Two load-bearing details: transition instructions carry the outgoing section's section_index (they occupy its time range), so replace_section regenerating the incoming section never orphans them; and the pass is idempotent via an extra_settings marker key, so re-running after a splice replaces rather than stacks transitions. Gating only trims end times (never reorders), so _guard_wash_occlusion in finalize_effects sees final geometry.

Decision 6 — color script is a deterministic plan post-pass. apply_color_script(plan, repetition_map) (new pipeline/color_script.py, or a pass in show-plan realization): anchor = the most frequent resolvable color across section palettes, injected into any section missing it; the chorus label's sections get a shared signature pair (their two most hue-distant colors, reused verbatim across occurrences — contrast_anchors in xlights_core.knowledge.colors is the existing hue-distance tool); the lowest-recurrence mid-song section (bridge heuristic) gets a complement pushed to the front. Runs right after the Director produces the plan and after any section redesign (the _redesign path in run.py:156–165). No LLM call.

Decision 7 — phrase dynamics ride existing value-curve helpers. In realize_section, beds/washes spanning ≥ 2 bars get a brightness curve shaped by the section's energy-arc slice: rising → brightness_ramp(lo, hi) (already imported at generate.py:18), falling → reversed, flat → none (today's behavior). 8-bar phrase starts on the downbeat grid get a one-beat swell on the bed via the accent layer's existing boost mechanism (_mk(..., boost=) in place_beat_accents). Only beds/washes — features and accents keep crisp levels. This generalizes the current single-case rule at generate.py:172 (_si ≥ 0.7 and span > 15000).

Touched modules: new pipeline/transitions.py, new pipeline/color_script.py, pipeline/generate.py (run the pass; phrase curves), pipeline/run.py/pipeline/regen.py (re-run after splices). Risk profile: additive passes; the blackout gate is the only piece that mutates existing instructions (trim-only, before finalize_effects).

3.4 The advisory metrics, end to end

New qa/musicality.py exposes evaluate(instructions, plan, brief) -> tuple[int, list[Finding]] (mirroring qa/variety.py's shape) and is wired into qa/__init__.evaluate outside the parts list that forms objective_score. Every finding carries objective=False and a namespaced metric (musicality:rhyme, musicality:dynamic-range, musicality:focus) so revision_log.source_of distinguishes them in the JSONL, the Judge sees them in judge_mod.render_input, and the human sees them in the attended checkpoint printout (run.py:90–95). Each metric emits a bounded number of findings (one per label/section, hard cap) to keep Judge input from getting noisy. Because they never enter objective_score, the loop's revert (REGRESS_MARGIN), stall (STALL_LIMIT), plateau, and skip (REFINE_SKIP_OBJECTIVE = 88) behaviors are untouched — but note the plateau signature at run.py:236 includes advisory_score; if advisory_score is later extended to blend musicality (open question below), plateau sensitivity changes slightly.

Roadmap cross-link: I8 Tier 0 defines the pixel-level versions of repetition-rhyme/dynamic-range from .fseq channel data, which — unlike these instruction-level metrics — could enter the objective gate. Landing F-A's instruction-level metrics first is deliberate: they are cheap, hermetic, and define the vocabulary; I8 can later corroborate them with rendered truth.

Ordered per tasks.md; each phase is its own branch/PR (task 4.1 — never commit to main directly).

5. Testing strategy

  • Hermetic-first, per repo convention: pyproject.toml sets addopts = "-m 'not live'"; all new tests run without xLights or API keys, using fake agents (the golden test drives run_pipeline end-to-end with TestModel agents and a fixed rich SongAnalysis).
  • Golden discipline: tests/test_golden_pipeline.py is the safety net for behavior-preserving refactors; each phase changes generation deliberately, so regenerate once per phase with XLO_REGEN_GOLDEN=1 and review the diff as part of the PR (carriers-per-label in P1; sparser verses in P2; boundary gating/risers in P3). Never regenerate to silence an unexplained diff.
  • Unit seams: section_identity, resolve_treatment, place_transitions, apply_color_script, and each qa/musicality metric are pure functions over pydantic models — table-driven tests, no mocks. The existing suites to extend: tests/test_escalation.py, test_peak_escalation.py, test_coverage_qa.py, test_refine.py, test_darkness.py, test_brief_editor.py.
  • Back-compat fixtures: load a cached pre-change creative_brief JSON (no treatment field) and assert validation + fallback resolution (spec scenario "Old cached plans still work").
  • Refine-loop integration: a hermetic _refine_loop run (injected fake qa/judge/regenerate as in test_refine.py) asserting advisory musicality findings appear in the record stream without moving objective_score, and that a splice + re-run of place_transitions leaves exactly one riser.
  • Live verify per phase (opt-in -m live plus the task-4.2 eyeball pass) before archiving.

7. Acceptance criteria

Phase 1 is done when:

  • ☐ Sections sharing a repetition_map label receive the same weave carrier, peak composite, and expanded-palette order; one-off sections keep index-keyed variety (tests 1.1–1.3).
  • ☐ The final occurrence of a recurring label lights ≥ as many groups as the first, with tighter accent stride and one extra layer when accent props exist, without exceeding clamp_layer_budget/clamp_hard_caps (test 1.4).
  • ☐ qa/musicality.py emits advisory repetition-rhyme findings; objective_score is provably unchanged by them (test 1.5).
  • ☐ Golden regenerated once, diff reviewed, pytest -m "not live" green (task 1.6).

Phase 2 is done when:

  • ☐ I3 has merged first (refine-loop decomposition + constants in tuning.py).
  • ☐ SectionPlan.treatment exists, defaulted; old cached briefs load and resolve treatments deterministically (tasks 2.1–2.2; spec "Old cached plans still work").
  • ☐ Each treatment's layer inventory matches the design matrix — layers withheld, not dimmed; a gesture section is a single carrier recipe on ≤2 groups (task 2.3; spec "A quiet verse is sparse, not dim").
  • ☐ A dark rest section produces no objective coverage error; a dark high-energy full section still does (task 2.4).
  • ☐ Dynamic-range and focus-budget advisory metrics behave per tests in 2.5; golden regenerated, suite green (2.6).

Phase 3 is done when:

  • ☐ Risers, blackout-before-drop, and sweep handoffs place from pure signal math; transition instructions carry the outgoing section_index; the pass is idempotent (task 3.1; spec "Regen keeps the riser" — exactly one riser after a splice).
  • ☐ The color script injects one anchor color into every section, gives all chorus-label sections a verbatim signature pair, and pushes a complement at the bridge — with no LLM call (task 3.3).
  • ☐ ≥2-bar beds/washes over rising/falling energy carry ramped brightness curves; accents/features keep crisp levels (task 3.4).
  • ☐ Golden regenerated, suite green (3.5).

Landing is done when:

  • ☐ Three phase PRs merged; before/after visual bundles reviewed per phase (tasks 4.1–4.2).
  • ☐ Treatment vocabulary + advisory metrics + golden-regeneration policy documented (task 4.3).
  • ☐ The three stale changes (add-refine-skip-high-objective, add-song-end-envelope-fade, add-targeted-section-regen) are archived.

8. Open questions

  • Bridge heuristic: LLM flag (bridge on SectionPlan) vs "lowest-recurrence mid-song section"? The change defaults to the heuristic; revisit if it misfires (design.md Open Questions).
  • Should treatment eventually gate the generator prompt per treatment (smaller prompts for gesture/rest)? A cost lever, explicitly out of scope here.
  • Should advisory_score (currently just variety, qa/__init__.py:32) blend the musicality score, or do the metrics stay findings-only? Blending changes the plateau signature at run.py:236 — recommend findings-only in phase PRs and a separate decision afterward.
  • Where exactly does the identity label thread — realize_section computing it internally from st.music_brief (matching the existing _rm fetch) vs an explicit parameter on section_carrier? Internal keeps call sites (run.py, regen.py) untouched.
  • Interaction with I7 (density/motion gap re-tuning): Phase 2 changes what "dense" should mean per section — re-run the corpus comparison after Phase 2 lands, not before.
