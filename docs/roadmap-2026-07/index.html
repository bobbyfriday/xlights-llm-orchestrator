Roadmap 2026-07 — per-item design docs

One thorough, standalone design document per roadmap item, written before the execution phase to capture as much design intent as possible. Each doc grounds its proposal in the current code (real paths, functions, constants) and ends with an implementation plan, testing strategy, risks, and acceptance criteria. The prioritization and sequencing live in the roadmap; these docs carry the depth.
