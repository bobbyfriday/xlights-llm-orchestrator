1. Why

The effects catalog and scene cookbook both name the matrix's real job — "the matrix is the storyteller" (xlights-scene-cookbook.md SC-08, citing Vol 3 §6) — and the craft roadmap's item 8 scopes the achievable slice: "generate Text effects on the matrix (section labels, lyric phrases, song title) — achievable with stock effects + no assets." Today the pipeline generates zero narrative content. The matrix (a real model in this layout — tests/fixtures/getModels.json lists "Matrix" alongside Arches / Mega Tree / House Outline) is treated as just another focal canvas: it gets washes, weave cells, the hero onset layer, and peak composite stacks via SEM_FOCAL, but never a single word — even when the brief has already picked out the song's featured lyric lines with timestamps and reasons (MusicBrief.featured_lyric_moments, music_brief.py:65).

The payoff-to-effort ratio is the best on the board (roadmap: "Highest visible payoff per unit work"): the lyric timing track already exists, the brief already contains curated lyric moments and the song's identity, the Text effect is stock and asset-free, and F-B provides the placement route. What's missing is one deterministic pass and a fistful of taste rules about when text should not appear.

2. Current state

2.1 The matrix in the layout and the group vocabulary

The layout-semantics spec classifies matrices as role MATRIX ("2D pixel surface capable of images/text", capability class 2D_SURFACE: "Supports pictures, text, video, shader-style effects" — xlights-layout-semantics-spec.md §2). Two consequences in code:

  • No dedicated SEM_ group. Spec §5: "Single-instance roles (MEGA_TREE, MATRIX, STAR, SIGN) do not need a group; the planner addresses the model directly." Accordingly layout_semantics._ROLE_GROUP (line 43) has no MATRIX entry, and pipeline/semantic_groups.py has no matrix constant.
  • It is focal. Spec §4 step 6 marks MATRIX props focal: true, so it lands in SEM_FOCAL (build_sem_groups, layout_semantics.py:75), and the render-order tiers treat any name containing "matrix" as the focal tier (_ORDER_TIERS, layout_semantics.py:122: lambda n: n == "SEM_FOCAL" or "matrix" in n.lower()) — rendered late, i.e. winning overlaps.

Targeting the model directly is legal end-to-end: place_preset's only target check is membership in await client.get_models() (editing.py:83), which includes plain models, and the emitter passes any EffectInstruction.target through. The one caveat: State.available_groups comes from targetable_groups() (pipeline/groups.py), which probes groups — the pass must resolve the matrix model name separately (see §3.2).

2.2 What already targets the matrix (the collision surface)

Because the matrix rides SEM_FOCAL, three existing layers can be live on it at any moment (generate.realize_section):

  • the hero onset layer — the melodic lead's onsets on HERO_GROUP = SEM_FOCAL (beats.place_beat_accents, semantic_groups.py:28);
  • the peak composite stack — curated_composite(_PEAK_COMPOSITES[...], [HERO_GROUP]) on peak sections (generate.py:196–199), 2 blended layers of Morph/Galaxy/etc.;
  • whatever washes/weave cells the Generator aims at SEM_FOCAL per the brief.

Any Text design that ignores this puts glyphs behind (or under) a kaleidoscope. §3.5 addresses it.

Line-level timing feeds the "Lyrics" reference timing track (timing.py:123–127) that F-C's phrases can cross-check against.

2.4 Text effect ground truth from the guides

xlights-effects-catalog.md §8 — Text: "Static or scrolling text rendering with font/size/movement controls. … Needs pixel density — minimum ~10–12 px of height for legible characters; test at distance. Group canvases spanning mismatched props mangle glyphs; keep Text on a single matrix or a carefully built group. Best on: Matrix only (practically)." Catalog rule (§9 #2): "Never place media effects (Text, Pictures, Video) on canvases under ~50px of resolution."

xlights-scene-cookbook.md SC — Masked Texture Reveal (lines 170–182): Text with "1 is Mask" over Bars/Plasma = "texture shows only inside glyphs … premium production value at near-zero render cost." Failure modes: "text too small for canvas density; busy effects behind small glyphs destroy legibility — bars beat plasma for thin fonts."

And Text is absent from the mined catalog (presets/looks.json has no Text key), so placement is impossible today — the F-B dependency.

3. Proposed design

3.1 Shape: a deterministic pass, LLM judgment upstream

New pipeline/matrix_text.py with one entry point, run inside generate_instructions after the per-section loop (beside place_triggers/key_moment_flashes, generate.py:256–258):

# PROPOSED
def place_matrix_text(st: State, matrix: str) -> list[EffectInstruction]:
    """Text moments on the matrix: song title at the intro, featured lyric phrases
    at their aligned times, an optional outro sign-off. Deterministic; content and
    times come from the brief (LLM judgment already spent there). Instructions
    carry direct_settings (F-B) and the OWNING section's section_index."""

Decision: deterministic pass, not a new Director field — recommended after weighing both.

OptionForAgainst
A. Director chooses via a new SectionPlan/KeyMoment field (e.g. KeyMoment.text) Matches "LLM chooses intent"; the Director could phrase-pick creatively (a word, not a line) Schema churn + prompt churn + cached-brief invalidation for judgment that is already in the brief: the panel's lyric analyst curated featured_lyric_moments with timestamps and a why. Asking the Director again duplicates a decision and adds a hallucination surface (text not in the song).
B. Deterministic pass over existing brief fields Zero schema change; content is grounded by construction (only strings that exist in featured_lyric_moments/identity can appear); testable as a pure function; consistent with the codebase's judgment-vs-realization split (the same reasoning as design decision 6 in improve-musicality: "no new Director round-trips … deterministic post-pass") Can't invent a clever caption; section labels as text would be literal ("VERSE 2") — mitigated by simply not using them (§3.3)

Recommendation: Option B now, with a forward-compatible escape hatch: if a KeyMoment of kind "lyric" ever gains a text field (one defaulted string — cheap to add later), the pass prefers it over its own selection. That keeps the door open for Director-chosen captions without blocking on prompt work.

3.2 Resolving the matrix target

    (case-insensitive), mirroring layout_semantics._ORDER_TIERS' focal test.

The pass needs model names, not group names: add a best-effort fetch beside the existing targetable_groups call in run_pipeline (run.py:386) — st.model_names = await client.get_model_names() wrapped in the usual except Exception posture — or pipe it through State. Never place Text on SEM_FOCAL (a group canvas spanning matrix + mega tree would "mangle glyphs" per the catalog). Longer-term this is exactly the manifest gap roadmap F-E fixes (spec §6's has_matrix/node-count data); until then the name heuristic matches the one already shipped in canonical_order.

3.3 Content selection: when text appears — and when it must not

Doctrine: do not caption the song. A matrix that talks all night is a chyron, not a showpiece; text is punctuation, like the white climax flash. Rules, in priority order, with an overall cap:

  1. Title card — identity.title (+ artist if it fits) once, in the intro section, static or slow scroll, ending ≥ 1 bar before the first downbeat-anchored section boundary. Skip if the intro is shorter than 8 s or the title is empty.
  2. Featured lyric phrases — each FeaturedLyricMoment whose span can be cross-checked against the aligned lyric lines (sa.lyrics["lines"]): snap the Text span to the matched line's start/end when a fuzzy match exists; if the brief's moment matches no aligned line, drop it (never show text at a guessed time). Cap: at most MAX_TEXT_MOMENTS = 4 lyric phrases per show, minimum TEXT_SPACING_MS = 20_000 apart, at most one per section, none inside the peak section (the peak belongs to the composite payoff — text would compete with the biggest gesture).
  3. Outro sign-off (optional, config-gated): a short "Merry Christmas"-class message only if the brief's key_moments contain a lyric-kind moment in the final section; otherwise nothing. v1 may ship without this.

Explicit non-sources: section labels ("CHORUS") — literal scaffolding, never audience-facing; full lyric captioning — out of scope by doctrine; anything not present verbatim in the brief/analysis — grounding rule, mirroring the Director prompt's "do NOT invent lyrics."

Instrumental behavior: with no featured_lyric_moments and no aligned lines, only the title card survives — acceptable and desirable (the roadmap's instrumental flag in director.render_input, director.py:24, is the same signal).

3.5 Interaction with the matrix's focal duties

The matrix is likely SEM_FOCAL's biggest member, so a Text moment coexists with hero onsets, washes, and (at the peak) composites. Design rules:

  • Render order already favors us: the focal tier renders late (canonical_order), and placing Text on the model row (not the group) means the model row's effects sit visually with the group rows resolved by patch_xsq_render_order at finalize. But layer blending is the real mechanism:
  • Text rides on top, masked or Max-blended. Emit Text with on_top=True (show_plan.py:78 — the punch-through mechanism triggers already use: top layer, exempt from clamp_layer_budget) so _top_layer places it above every overlapping layer, plus T_CHOICE_LayerMethod: "Max" in extra_settings so glyph pixels add over the busy background instead of black-boxing it. v2 option: the cookbook's "1 is Mask" recipe (Text as mask over a Bars layer) for the premium look — defer until the plain version is verified live.
  • Calm the background during text. During each Text span, the pass dims concurrent matrix-targeted non-text instructions (multiply their C_SLIDER_Brightness by ~0.4, floor via brightness_setting) — the catalog's Faces rule ("surrounding props dim to ≤30% so the singer owns focus") applied to glyphs. Never dim other props: only the canvas behind the letters.
  • Keep out of the peak (§3.3 rule 2) so Text never collides with the curated composite on HERO_GROUP.
  • QA: Text is not in rules.FEATURES (one-feature-at-a-time) nor ENERGY_BAND — unconstrained objectively; add one advisory in qa/variety.py-style if text moments exceed the cap (belt-and-braces against future authors).

3.6 Tagging, regen, and idempotence

Each Text instruction carries the owning section's section_index, so replace_section (refine.py:48–54) drops-and-recreates them correctly when that section regenerates — which means the pass must re-run after refine-loop/xlo regen splices, exactly like improve-musicality's transitions pass (design decision 5 there), and be idempotent the same way: a marker key in extra_settings (e.g. X_MatrixText=1 — stripped before emit or tolerated as an unknown key xLights ignores; verify in the F-B probe) lets the pass replace its own prior output instead of stacking. Sequence the implementation after improve-musicality Phase 3 lands place_transitions, and copy its splice-hook pattern.

4. Implementation plan

  1. Land F-B first (hard dependency): build_text_settings, direct_settings field, emitter branch, frozen Text template.
  2. Matrix discovery: find_matrix() + State.model_names plumbing (best-effort fetch in run_pipeline step 1 area, near targetable_groups); hermetic test with the fixtures' getModels.json list.
  3. Selection engine: select_text_moments(brief, sa, plan) -> list[TextMoment] (pure): title-card rule, featured-phrase cross-check against aligned lines, caps/spacing/peak-exclusion, instrumental degradation. Table-driven tests: vocal song with 6 candidate moments → 4 placed, spacing enforced; moment with no aligned line → dropped; instrumental → title only; no matrix → [].
  4. Realization: place_matrix_text(st, matrix) building instructions (direct_settings, on_top=True, Max blend, lightest-color palette, static-vs-scroll decision, background-dim adjustments); wire into generate_instructions after the section loop and into the refine/regen splice path beside (or inside) the transitions re-run.
  5. Idempotence + regen tests: re-running the pass after replace_section yields exactly the same text moments (no stacking); regenerating a section that owns a text moment recreates it.
  6. Golden: the fixture brief has no lyrics, so expected golden impact is nil or a title card only — regenerate with XLO_REGEN_GOLDEN=1 if it changes, review the diff.
  7. Live verify: run a vocal reference song end-to-end; eyeball the review bundle stills at each text moment (legibility, scroll-once timing, background dim); check the xLights log for ApplySetting errors from the Text template.
  8. Docs: note the text doctrine (punctuation, caps, non-sources) in docs/usage.md's features section and cross-link the craft roadmap item as closed.

5. Testing strategy

  • Hermetic-first (pytest -m "not live", fake agents): everything in steps 2–5 is pure-function or fake-client testable. New tests/test_matrix_text.py covering selection (caps, spacing, peak exclusion, grounding drops, instrumental, no-matrix), realization (settings keys via parse_settings, on_top/blend/palette), and splice idempotence via the test_refine.py-style injected loop.
  • Fixture: extend a copy of the golden's SongAnalysis with lyrics.lines (word spans) + a brief with featured_lyric_moments, so selection tests run against realistic timing data.
  • Settings validation: every emitted direct_settings round-trips through parse_settings/serialize_settings and contains E_TEXTCTRL_Text equal to the sanitized source string (F-B ring-1).
  • Live (opt-in -m live): place the title card + one scrolling phrase on the real Matrix via validate_direct-style scratch sequence; human eyeball at render.

7. Acceptance criteria

  • ☐ On a vocal reference song with aligned lyrics, the show contains a title card in the intro and ≤ 4 featured lyric phrases, each snapped to its aligned line's span, ≥ 20 s apart, none in the peak section.
  • ☐ On an instrumental song, at most the title card appears; with no matrix in the layout, the pass no-ops with a logged reason.
  • ☐ Every Text instruction targets the matrix model (never a group), carries direct_settings that round-trip the parser, on_top=True, Max blend, and the section's lightest palette color.
  • ☐ Scrolling phrases traverse exactly once within their aligned span (speed math test); static text fits the probed width.
  • ☐ Concurrent matrix-targeted instructions are dimmed during text spans; non-matrix props are untouched.
  • ☐ Regenerating a section that owns a text moment (refine splice or xlo regen) reproduces exactly one copy of it.
  • ☐ Hermetic suite green (pytest -m "not live"); golden regenerated-and-reviewed only if it changed; live eyeball checklist done on a real render.

8. Open questions

  • Matrix pixel dimensions: probe via get_model at run time vs a one-line layout constant until F-E's §6 manifest exists? (Probe is ~free and self-maintaining; recommend probe with constant fallback.)
  • Should the masked-texture recipe (Text "1 is Mask" over Bars — cookbook SC) be v1 or v2? Recommend v2: it doubles the layer choreography and the plain bright-on-dim version must be proven legible first.
  • Outro sign-off content: config option (xlo flag / brief field) or omit entirely in v1?
  • Later Director hook: add KeyMoment.text so the LLM can hand-pick a word ("HALLELUJAH" not the whole line)? Cheap and back-compatible; decide after watching v1 output.
  • Does the white key_moment_flashes flash on SEM_ALL visually stomp a concurrent text moment (the matrix is in SEM_ALL)? If live verify says yes, exclude flash overlap in selection.
